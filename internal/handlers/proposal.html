Feature Catalogue

01

Real-time Conversation Inbox

Live WebSocket inbox

New messages, typing indicators, and status updates push to every connected agent within milliseconds — no polling.

Multi-account WhatsApp

Connect any number of WhatsApp Business Cloud accounts. Route inbound traffic to the right team automatically.

All media types

Images, video, audio, documents, stickers — full preview in-app, transcoded server-side when needed (Chrome webm → Meta ogg).

Voice notes

In-browser recording with waveform preview. Auto-converted to the format Meta accepts before upload.

Reactions & replies

Full emoji reaction support and threaded quote-replies, mirroring native WhatsApp UX.

Interactive buttons

Send reply buttons (up to 10 as a list), URL CTAs, and phone-call buttons. WhatsApp-renders them natively on the recipient's device.

Templates & WhatsApp Flows

Sync Meta-approved templates, fill placeholders inline, and trigger multi-screen WhatsApp Flows for richer interactions.

Canned responses

Per-team snippet library with variable substitution and button payloads. Saves agents 20+ minutes per shift.

Internal notes

Private notes attached to conversations and contacts. Visible to colleagues, never to the customer.

Conversation context panel

Contact details, tags, custom fields, message history, and audit trail — all one click away from the chat.

02

No-Code Chatbot Builder

Visual flow editor

Drag-and-drop canvas. Connect nodes by dragging from output to input — no code, no YAML.

14 node types

Message, Buttons, Condition, Prompt, API Call, Webhook, Set Variable, AI Response, Transfer, Goto Flow, Timing, WhatsApp Flow, Start, End.

Live preview & debugger

Simulate any flow without messaging a real customer. Step-through timeline shows every variable, branch, and API response.

AI-powered responses

Drop an "AI Response" node anywhere. Pulls from per-org context documents, answers in the customer's language.

Keyword triggers

Regex or exact match. Auto-start a flow when the customer says "menu", "support", or anything you define.

External API calls

Hit any HTTP endpoint mid-flow. Pull order status from your ERP, push leads to your CRM, all without writing a backend.

Business hours routing

"In-hours" vs "out-of-hours" branching based on per-day schedules. Hand off to a human only when one is online.

Sub-flows & variables

Goto-flow keeps the conversation state. Variables flow forward; reusable sub-flows mean no copy-paste.

08

Security & Compliance

AES-256 token encryption

WhatsApp access tokens and webhook secrets are encrypted at rest. Decryption keys live in env vars, never the DB.

bcrypt + JWT

Passwords hashed with bcrypt (cost 10). Tokens live in httpOnly, Secure, SameSite cookies — XSS-resistant.

Rate limiting

Per-IP login/register limiters and per-user API throttle. Redis-backed fixed-window counters.

CORS & origin checks

Strict allow-list. Webhook signatures verified against Meta's app secret. CSRF-safe by default.

Immutable audit log

Append-only. Records who did what, when, from where, and the before/after diff for every mutation.

Self-hosted

Runs entirely in your own infrastructure — VPC, on-prem, anywhere. Customer messages never traverse a third-party SaaS.

Deployment & Operations

One Docker image, one command.

We ship as a single Alpine-based Docker image. The Go binary embeds the compiled frontend, so there's no separate web server to manage.

docker-compose.yml · 3 services ~ 30s to start 
services:
  app:
    image: banglab2bb2c:latest
    ports: ["8080:8080"]
    environment:
      BANGLAB2BB2C_DATABASE_HOST: db
      BANGLAB2BB2C_REDIS_HOST: redis
      BANGLAB2BB2C_APP_ENCRYPTION_KEY: ${ENC_KEY}
      BANGLAB2BB2C_JWT_SECRET: ${JWT}
    depends_on: [db, redis]
    restart: unless-stopped

  db:
    image: postgres:17-alpine
    volumes: [db-data:/var/lib/postgresql/data]

  redis:
    image: redis:7-alpine

# That's it. Migrations auto-run on boot.
# Default admin user is seeded from env vars.

Health monitoring

GET /health returns the build version + db / redis status. Wire into your monitoring of choice.

Coolify-ready

One-click Coolify deployment with automatic Let's Encrypt TLS via Traefik. Also works with Kubernetes, Nomad, or plain Docker.

Backups

PostgreSQL dump + Redis RDB are sufficient for a full restore. We document the procedure and provide a shell script.

Frequently asked

Things you'll probably ask in the discovery call.

Who controls the Meta WhatsApp Business account?
You do. The platform connects to your Meta Business account using credentials you own. We never have access to your tokens or customers' messages. If you ever stop using the platform, your Meta account stays exactly as it was — only the integration goes away.
What does it cost to send a WhatsApp message?
Meta charges per-conversation, not per-message. Their pricing varies by country and conversation category (marketing / utility / authentication / service). Service conversations initiated by the customer are typically free for the first 1,000 per month per WhatsApp Business Account. Marketing template sends are charged. The platform's Meta Insights tab surfaces the current price-per-conversation live.
Can we customise the platform to match our brand and processes?
Yes — the source license gives your engineers full editing rights. Our recommended approach is to keep the upstream untouched and apply your customisations as a small overlay so future updates remain easy to pull in. We can advise on this.
How long does Meta Business approval typically take in Bangladesh?
Display name verification is usually 1-3 business days. Business verification can take 5-15 business days if Meta requests supporting documents (utility bill, trade license, etc.). We've shepherded several Bangladeshi businesses through this and can hand you a checklist that minimises back-and-forth.
What's the realistic minimum infrastructure to run this?
For up to ~50 agents and modest campaign volume: a single 4 vCPU / 8 GB VPS with a managed PostgreSQL (or a second small VPS for the database) and a small Redis instance. Storage scales with media volume — most customers see 5-20 GB / month.
What happens if the WhatsApp Cloud API changes?
Meta versions their API; we pin a version per account. When Meta deprecates an endpoint they typically give 12-18 months of overlap. Under the Managed plan we ship the upgrade before deprecation. Under the License / Turn-key plans we offer a discounted upgrade SKU.
